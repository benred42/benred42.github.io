Learning the Ropes:
Matrix Math

Early on at the Iron Yard, I built a library to perform various vector/matrix math operations, first from a function-based approach and then from a class-based approach. Not only did I learn a lot of the math that powers the popular NumPy library that I worked with later, but I gained a greater depth of understanding on object-oriented principles and how powerful OO programming can be.

(GitHub for function-based approach, GitHub for class-based approach)

Technologies: Python 3 and standard libraries, IPython Notebook

Diving into Data:
Traffic Simulation

One of the focuses of my Iron Yard track was on data science and one of the most fun and interesting data projects I worked on was a traffic simulation. Given some basic assumptions and a degree of randomness, I was able to simulate the flow of traffic under four different scenarios, resulting in some compelling visualizations. It was one of the first times I really applied my skills to a real-world problem and I greatly enjoyed it.

(GitHub, IPython Notebook)

Technologies: Python 3, IPython Notebook, NumPy, Matplotlib, Seaborn

Learning with Machines:
Honey Nut Clustering

One of several machine learning projects I worked on at the Iron Yard, this project required me to cluster cereals based on their nutrition facts. One of the challenges posed by this project in particular was determining, for a number of clusters produced, if those clusters were meaningful in any real world sense. It was an interesting challenge and I learned about both the power of machine learning to cluster and classify data and the necessity to ensure your results are trustworthy.

(GitHub, IPython Notebook)

Technologies: Python 3, IPython Notebook, Pandas, SciKit-Learn, NumPy, Matplotlib, K-Means Clustering

Working on Web Apps:
URLyBird

URLy bird was the second Django application I worked on. It allows users to bookmark links, providing them with a shortened URL to that link. Additionally, any time that shortened link is used, it records the event so the user can keep track of their link traffic. Beyond the actual shortening of the URL itself, one of the biggest technical challenges for this project was capturing meaningful data every time a shortened URL was used. This was also one of the first projects I worked on where UI/UX design came into play, which was a new enjoyable challenge for me.

(GitHub, Live)

Technologies: Python 3, Django, HTML5, Bootstrap, Django Rest Framework, PostgreSQL, Heroku

Bringing it all Together:
Predictive Inventory Management

My final project at the Iron Yard, I worked on PIM with a team of two other Python students and two Front-End students. We built an application that solves the tedious and difficult task of spa inventory management. PIM looks ahead to the upcoming appointments and determines how much of various products will be used so it can warn a spa owner ahead of time when their inventory will be low. We incorporated numerous technologies and features, such as UPC scanning and form completion via an API, the ability to send order emails to suppliers directly from the application, and several useful interactive graphical reports. Working on PIM helped give me a feel for working with a team on a larger project and I learned a great deal about the technologies involved and how to stretch myself to reach technical goals that are beyond what I currently know how to accomplish.

(GitHub, Live)

Technologies: Python 3, Django, HTML5, CSS3, JavaScript, D3, jQueryUI, FullCalendar, PostgreSQL, Heroku, Git
